01. Arquitetura ReAct
Reason + Act: como o agente pensa e age

ReAct é um padrão de prompt que instrui o LLM a intercalar raciocínio e ação. Cada ciclo tem 3 fases obrigatórias antes de chegar na resposta final.

O ciclo ReAct

FaseO que aconteceVisível no terminal
ThoughtO LLM raciocina sobre o que fazerThought: Preciso calcular...
ActionEscolhe e chama uma toolAction: calcular
ObservationRecebe o retorno da toolObservation: 17997.0
Final AnswerSintetiza a resposta finalFinal Answer: O total é...
Por que isso importa

O agente não adivinhe — ele raciocina em voz alta antes de agir. Isso torna o comportamento auditavel e previsível.

Demonstração
02. Demo multi-tool
Agente com 3 tools respondendo uma pergunta composta

O professor vai rodar um agente com 3 tools diferentes: pesquisa de produto, cálculo de desconto e conversão de moeda. A pergunta exige as 3 em sequência.

Pergunta usada na demo

"Qual o preço do iPhone 15 com 10% de desconto, convertido para USD?"

Por que essa pergunta

Ela força o agente a usar as 3 tools em ordem: pesquisar_produto → calcular_desconto → converter_moeda. Nenhuma if/else no código decide isso.

Demonstração
03. Código com 3 tools
Agente completo usado na demonstração
agente_react_demo.py
import os from dotenv import load_dotenv from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain.tools import tool from langchain import hub load_dotenv() llm = ChatOpenAI( base_url="https://api.llama-api.com", api_key=os.getenv("LLAMA_API_KEY"), model="llama3.1-70b" ) @tool def pesquisar_produto(nome: str) -> str: """Busca preco e disponibilidade de um produto pelo nome.""" catalogo = { "iphone 15": "R$ 5.999,00 - Em estoque", "samsung s24": "R$ 4.799,00 - Em estoque", "pixel 8": "R$ 4.299,00 - Ultimas unidades" } return catalogo.get(nome.lower(), "Produto nao encontrado no catalogo") @tool def calcular_desconto(preco: float, percentual: float) -> str: """Calcula o preco final apos aplicar um percentual de desconto.""" desconto = preco * (percentual / 100) final = preco - desconto return f"Preco original: R$ {preco:.2f} | Desconto: R$ {desconto:.2f} | Final: R$ {final:.2f}" @tool def converter_moeda(valor: float, moeda: str) -> str: """Converte um valor em BRL para USD, EUR ou GBP.""" taxas = {"USD": 0.20, "EUR": 0.18, "GBP": 0.16} moeda = moeda.upper() if moeda in taxas: convertido = valor * taxas[moeda] return f"R$ {valor:.2f} = {moeda} {convertido:.2f}" return "Moeda nao suportada. Opcoes: USD, EUR, GBP" tools = [pesquisar_produto, calcular_desconto, converter_moeda] prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=6) resposta = executor.invoke({ "input": "Qual o preco do iPhone 15 com 10% de desconto, convertido para USD?" }) print("\nResposta final:", resposta["output"])
Demonstração
04. Ciclo Thought/Action/Observation
O que cada linha do output significa
  • Thought: preciso saber o preço do iPhone 15 — o agente verbaliza sua intenção antes de agir. Esse texto não chega ao usuário final.
  • Action: pesquisar_produto / Action Input: "iphone 15" — o LLM escolheu a tool correta e passou o argumento certo, sem instrução explícita.
  • Observation: R$ 5.999,00 - Em estoque — retorno da tool injetado de volta no contexto do LLM para o próximo raciocínio.
  • max_iterations=6 — limite de ciclos para evitar loop infinito em casos de ambiguidade. Ajuste conforme a complexidade da tarefa.

05. Instrução para você
Crie um agente com tools do seu contexto profissional

Use o template da seção 07 e substitua as tools por funções relevantes para o seu trabalho. A descrição das tools é o que o LLM usa para decidir quando chamá-las — seja específico.

Exemplos de tools por área

  • Vendas — consultar_crm, calcular_comissao, verificar_estoque
  • Financeiro — buscar_cambio, calcular_roi, converter_unidade
  • RH — consultar_politica, calcular_ferias, buscar_beneficio
  • Operações — verificar_prazo, calcular_frete, buscar_fornecedor
Atenção

A descrição da tool (o docstring) precisa ser clara e específica. O LLM lê esse texto para decidir quando usar a tool. Uma descrição vaga resulta em chamadas erradas.

Atividade Prática
06. Passo a passo
Como montar seu agente personalizado
1
Abra o arquivo da Aula 01. (1 min)
Use o agente.py que você criou como ponto de partida.
2
Defina 2 ou 3 tools. (5 min)
Pense em funções que respondem perguntas do seu dia a dia. Use o decorator @tool e escreva docstrings claras.
3
Monte uma pergunta que exija as 3 tools. (2 min)
Construa uma pergunta que o agente só consiga responder usando todas as tools em sequência.
4
Rode com verbose=True. (5 min)
Acompanhe o ciclo Thought → Action → Observation no terminal.
5
Ajuste as descrições. (5 min)
Se o agente escolheu a tool errada, melhore o docstring. Teste novamente.

07. Código para adaptar
Template com 3 tools para personalizar
agente_meu_contexto.py
import os from dotenv import load_dotenv from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain.tools import tool from langchain import hub load_dotenv() llm = ChatOpenAI( base_url="https://api.llama-api.com", api_key=os.getenv("LLAMA_API_KEY"), model="llama3.1-70b" ) # TOOL 1: substitua pela sua logica de consulta @tool def minha_tool_1(entrada: str) -> str: """Descreva claramente o que esta tool faz e quando deve ser usada.""" return f"Resultado da tool 1 para: {entrada}" # TOOL 2: substitua pela sua logica de calculo @tool def minha_tool_2(valor: float) -> str: """Descreva claramente o que esta tool faz e quando deve ser usada.""" return f"Resultado da tool 2 para valor: {valor}" # TOOL 3: substitua pela sua logica de conversao/busca @tool def minha_tool_3(parametro: str) -> str: """Descreva claramente o que esta tool faz e quando deve ser usada.""" return f"Resultado da tool 3 para: {parametro}" tools = [minha_tool_1, minha_tool_2, minha_tool_3] prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=8) # TROQUE pela sua pergunta que exige as 3 tools resposta = executor.invoke({ "input": "Sua pergunta aqui — deve exigir as 3 tools em sequencia." }) print("\nResposta:", resposta["output"])

08. Desafio extra
Adicione memória de conversa ao seu agente

O agente atual não lembra de interações anteriores. Para o desafio, adicione ConversationBufferMemory e faça duas perguntas relacionadas em sequência.

Snippet — adicionando memória
from langchain.memory import ConversationBufferMemory from langchain.agents import create_react_agent, AgentExecutor # Adicione ao executor memory = ConversationBufferMemory( memory_key="chat_history", return_messages=True ) executor = AgentExecutor( agent=agent, tools=tools, memory=memory, verbose=True ) # Primeira pergunta executor.invoke({"input": "Qual o preco do iPhone 15?"}) # Segunda pergunta referencia a primeira executor.invoke({"input": "E com 10% de desconto, quanto fica em USD?"})
Parabéns
