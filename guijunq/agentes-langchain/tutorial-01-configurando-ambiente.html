Introdução
01. O que são LangChain e Llama API Cloud
Dois frameworks complementares para construir agentes

LangChain

Framework open-source em Python para construir aplicações com LLMs. Fornece abstrações para chains, memory, agentes e tools — você orquestra o fluxo sem se preocupar com os detalhes de cada modelo.

  • Agentes — dão autonomia ao LLM para decidir quais ações tomar
  • Tools — funções Python que o agente pode chamar
  • Memory — mantém contexto entre interações

Llama API Cloud

API hospedada pela Meta para os modelos Llama 3.x. Interface compatível com OpenAI — qualquer SDK que usa OpenAI funciona com Llama API trocando apenas a base_url e a chave.

  • Modelos disponíveis — Llama 3.1 8B, 70B, 405B e Llama 3.3 70B
  • Velocidade — infraestrutura otimizada para inferência rápida
  • Custo — mais barato que GPT-4 para volumes altos
Por que usar juntos

LangChain orquestra a lógica do agente. Llama API executa o raciocínio. Você escreve o fluxo uma vez e pode trocar de modelo a qualquer hora.

Demonstração
02. O que o professor vai mostrar
Setup completo e primeiro agente rodando ao vivo

O professor vai executar ao vivo todo o fluxo de configuração e rodar um agente com duas tools: uma calculadora e um conversor de moeda.

Sequência da demo

  • Criar conta em cloud.llamaapi.com e gerar a API key
  • Instalar dependências em um ambiente virtual Python
  • Escrever o agente com LangChain e conectar à Llama API
  • Rodar ao vivo com verbose=True para ver o raciocínio
Dica de execução

Deixe o terminal com verbose=True visível durante a demo. Os alunos vêem o agente pensar em tempo real — isso é o que torna o conceito concreto.

03. Código da demo
Instalação e agente completo usado pelo professor
Instalação
pip install langchain langchain-openai python-dotenv langchainhub
agente_demo.py
import os from dotenv import load_dotenv from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain.tools import tool from langchain import hub load_dotenv() # Conecta LangChain na Llama API Cloud llm = ChatOpenAI( base_url="https://api.llama-api.com", api_key=os.getenv("LLAMA_API_KEY"), model="llama3.1-70b" ) # Define as tools do agente @tool def calcular(expressao: str) -> str: """Calcula uma expressao matematica simples. Ex: '10 * 5 + 3'""" try: return str(eval(expressao)) except Exception as e: return f"Erro: {e}" @tool def converter_moeda(valor: float, moeda: str) -> str: """Converte um valor em BRL para outra moeda. Moedas: USD, EUR, GBP""" taxas = {"USD": 0.20, "EUR": 0.18, "GBP": 0.16} moeda = moeda.upper() if moeda in taxas: resultado = valor * taxas[moeda] return f"R$ {valor:.2f} = {moeda} {resultado:.2f}" return "Moeda nao suportada. Use: USD, EUR ou GBP" # Cria o agente ReAct tools = [calcular, converter_moeda] prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # Executa resposta = executor.invoke({ "input": "Quanto custa 3 iPhones a R$ 5.999 cada, convertido para USD?" }) print("\nResposta:", resposta["output"])
Arquivo .env

Crie um arquivo .env na raiz do projeto com: LLAMA_API_KEY=sua_chave_aqui

04. O que observar na demo
3 padrões visíveis no output do agente
  • Ciclo Thought → Action → Observation — o agente imprime seu raciocínio antes de cada ação. Isso é o padrão ReAct em operação.
  • Seleção automática de tool — o LLM decide sozinho quando usar calcular vs converter_moeda. Nenhuma lógica if/else no seu código.
  • Composição de tools — para responder a pergunta, o agente usa calcular primeiro (3 × 5.999) e depois converter_moeda com o resultado.

06. Passo a passo
5 passos para ter o agente rodando
1
Crie um ambiente virtual. (1 min)
No terminal: python -m venv venv e depois source venv/bin/activate (Mac/Linux) ou venv\Scripts\activate (Windows).
2
Instale as dependências. (2 min)
Copie e cole o comando de instalação da seção 03. Aguarde o pip terminar.
3
Crie o arquivo .env. (1 min)
Na pasta do projeto, crie .env com sua chave: LLAMA_API_KEY=sua_chave_aqui
4
Cole o código. (2 min)
Crie agente.py, cole o código da seção 07 e substitua a pergunta por algo do seu contexto profissional.
5
Execute e observe. (5 min)
Rode python agente.py e acompanhe o raciocínio do agente no terminal.

07. Código para adaptar
Template com sua chave e uma tool para personalizar
agente.py — sua versão
import os from dotenv import load_dotenv from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain.tools import tool from langchain import hub load_dotenv() llm = ChatOpenAI( base_url="https://api.llama-api.com", api_key=os.getenv("LLAMA_API_KEY"), model="llama3.1-70b" ) @tool def calcular(expressao: str) -> str: """Calcula uma expressao matematica simples.""" try: return str(eval(expressao)) except Exception as e: return f"Erro: {e}" # PERSONALIZE AQUI: adicione uma tool do seu contexto @tool def minha_tool(entrada: str) -> str: """Descreva o que sua tool faz. Seja claro — o LLM usa isso para decidir quando chamar.""" # Implemente sua logica aqui return f"Resultado para: {entrada}" tools = [calcular, minha_tool] prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # TROQUE pela sua pergunta resposta = executor.invoke({ "input": "Escreva sua pergunta aqui — pode ser algo do seu trabalho." }) print("\nResposta:", resposta["output"])
Atividade Prática
08. Validando a configuração
Como confirmar que tudo está funcionando

Sinais de sucesso

  • Terminal mostra linhas começando com Thought:, Action: e Observation:
  • Resposta final impressa após Final Answer:
  • Sem erros de autenticação (401) ou rate limit (429)
Erros comuns

AuthenticationError: verifique se o arquivo .env está na pasta certa e se a chave está correta. ModuleNotFoundError: confirme que o venv está ativado antes de rodar.
