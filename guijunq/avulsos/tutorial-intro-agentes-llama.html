01. O que é um Agente de AI
A diferença entre um chatbot e um agente

Um chatbot responde perguntas com base no que sabe. Um agente pode executar ações no mundo real — chamar APIs, ler arquivos, fazer cálculos — e usar os resultados para formular a resposta.

Chatbot vs Agente

ChatbotAgente
Acesso a dadosApenas treinamentoTempo real via tools
AçõesSó textoExecuta funções
RaciocínioUma passagemMúltiplos ciclos
Caso de usoFAQ, suporteAutomação, análise
Mudança de paradigma

Você não programa o que fazer — você descreve as ferramentas disponíveis. O LLM decide como usá-las.

02. Demo ao vivo
Agente com Tool Calling nativo da Llama API

O professor vai usar a Llama API diretamente (sem LangChain) para mostrar como o Tool Calling funciona na camada mais básica — antes de qualquer framework.

O que a demo mostra

  • Definição de tool em JSON Schema — como o modelo "vê" a ferramenta
  • Resposta do modelo com tool_calls quando decide agir
  • Execução da tool e injectando o resultado de volta
  • Resposta final do modelo após receber o resultado
Demonstração
03. Tool Calling explicado
Código da demo com Tool Calling nativo
tool_calling_demo.py
import os, json from openai import OpenAI # SDK OpenAI funciona com Llama API client = OpenAI( base_url="https://api.llama-api.com", api_key=os.getenv("LLAMA_API_KEY") ) # Define a tool em JSON Schema tools = [{ "type": "function", "function": { "name": "buscar_cotacao", "description": "Busca a cotacao atual de uma moeda em relacao ao BRL", "parameters": { "type": "object", "properties": { "moeda": { "type": "string", "description": "Codigo da moeda: USD, EUR, GBP, etc." } }, "required": ["moeda"] } } }] # Funcao Python que executa a tool (dados fictícios para demo) def buscar_cotacao(moeda: str) -> dict: cotacoes = {"USD": 5.15, "EUR": 5.60, "GBP": 6.50} if moeda.upper() in cotacoes: return {"moeda": moeda.upper(), "valor_brl": cotacoes[moeda.upper()]} return {"erro": "Moeda nao encontrada"} # 1a chamada: pergunta ao modelo messages = [{"role": "user", "content": "Quanto custa 1 dolar hoje em reais?"}] resposta = client.chat.completions.create( model="llama3.1-70b", messages=messages, tools=tools ) # Verifica se o modelo quer usar uma tool msg = resposta.choices[0].message if msg.tool_calls: tool_call = msg.tool_calls[0] args = json.loads(tool_call.function.arguments) resultado = buscar_cotacao(args["moeda"]) # Injeta o resultado e chama novamente messages.append(msg) messages.append({ "role": "tool", "tool_call_id": tool_call.id, "content": json.dumps(resultado) }) resposta_final = client.chat.completions.create( model="llama3.1-70b", messages=messages ) print(resposta_final.choices[0].message.content)
Demonstração
04. O que observar
3 momentos-chave da demo
  • O modelo decide — o Llama retorna finish_reason: "tool_calls" quando quer usar a tool. Não é você que decide.
  • JSON Schema é o contrato — a descrição dos parâmetros em JSON é o que o modelo lê para saber como chamar a ferramenta. Uma descrição ruim = chamadas erradas.
  • Duas chamadas à API — a primeira pede ao modelo que decida, a segunda entrega o resultado. Agentes mais complexos fazem isso em loop até chegar na resposta final.

08. Próximos passos
O que aprender depois desta aula

Caminho sugerido

  • LangChain — framework que abstrai o loop de Tool Calling e adiciona memory, chains e agentes mais complexos
  • Múltiplas tools — adicione 2-3 tools e perguntas que exijam todas em sequência
  • APIs reais — substitua as funções mock por chamadas ao seu CRM, ERP ou qualquer API interna
  • Streaming — use stream=True na API para respostas progressivas
Conclusão

Você acabou de construir um agente de AI funcional com apenas 40 linhas de Python. O LLM decide, a tool executa, você controla o fluxo.
