Introdução
03. Conceito — Claude com contexto carregado
A diferença entre o chat comum e o Project com seus arquivos

Quando você usa o Claude no chat comum, toda conversa começa do zero. Você precisa explicar quem é, o que faz, qual a empresa, quais as prioridades — a cada nova análise. O Claude é brilhante, mas é como um consultor novo: precisa de briefing inteiro toda vez.

Um Claude Project resolve isso de forma permanente. Você configura o contexto uma única vez — instruções (System Prompt) + arquivos de contexto — e o Claude carrega tudo automaticamente em cada nova conversa dentro do Project. No nosso caso, os arquivos são as suas notas do vault Obsidian, então o Claude conhece a sua rede de pensamento.

💡 Analogia simples

Chat comum do Claude = contratar um consultor novo a cada conversa. Excelente, mas você briefa do zero toda vez.

Claude Project = ter um analista fixo que já conhece seu contexto. Você pergunta, ele responde com profundidade, sem precisar explicar nada.

Claude no chat comum
"Analise as decisões estratégicas do meu negócio…"
Claude responde com framework genérico — não sabe sua empresa, seu segmento, suas notas, suas decisões anteriores. Você gasta 30% da conversa só dando contexto. Resultado: análise rasa.
Claude no Project com seu vault
"Analise os arquivos do meu vault e me devolva os padrões"
Claude já tem os 3 arquivos .md do seu segundo cérebro carregados. Reconhece os links [[bidirecionais]], entende sua linha de pensamento, e te devolve análise no seu vocabulário. Resultado: insight personalizado e acionável.

💡 Por que o prompt tem essa estrutura

O prompt pede ao Claude 3 coisas em ordem: ler a rede (entender os links), sintetizar (padrões + contradições + perguntas) e produzir (output executivo). Essa ordem é importante porque força o Claude a entender antes de gerar — análise sólida antes da produção. Sem isso, o output sai genérico.

06. O que observar durante a demonstração
4 detalhes que distinguem o Project do chat comum
  • Arquivos abertos autonomamente. O Claude no Project abre os arquivos .md autonomamente — você não precisa colar conteúdo no chat. Os arquivos ficam disponíveis em todas as conversas do Project.
  • Sintaxe de links reconhecida. O Claude reconhece a sintaxe dos links [[bidirecionais]] mesmo sem você explicar — ele entende que [[Feedback de cliente: sabor da creatina vegetal]] se refere ao arquivo correspondente.
  • Vocabulário próprio. O output usa o vocabulário da Camila — "creatina vegetal", "Série A", "magnésio" — não framework genérico de consultoria. Isso só acontece porque o contexto está no Project.
  • Tudo em uma mensagem. O output cabe em uma única mensagem do Claude — confirmando que um prompt bem estruturado entrega análise + síntese + produção em uma só chamada, importante para quem usa o plano gratuito.

💡 Por que salvar o output de volta no vault

O segundo cérebro só funciona se for um loop: o que você captura vira insumo → o Claude processa → o output volta como nova nota → e essa nota é insumo da próxima conversa. Salvar o output no vault (pasta Resources) é o que faz o sistema acumular inteligência ao longo do tempo. Sem esse passo, você gera output, fecha o Claude, e perde tudo na próxima conversa.

PASSO 07Analise o output do Claude

Aguarde o Claude processar (tipicamente 30 a 60 segundos). Quando o output chegar, leia com atenção. Você deve ver 4 blocos: Padrões, Contradições, Perguntas em aberto e Primeiro output executivo.

O que validar:

  • O Claude usou o vocabulário das suas notas (não jargão genérico)?
  • Os links [[bidirecionais]] foram identificados e seguidos?
  • As perguntas em aberto te fazem pensar em algo que você não tinha verbalizado?
  • O output executivo é acionável — você pode usar HOJE?
Resposta do Claude na tela — os 4 blocos da análise visíveis (Padrões, Contradições, Perguntas, Output)
⚠ Se o output ficou genérico

Isso geralmente significa que o System Prompt está raso ou que faltam variáveis preenchidas. Volte ao Passo 03, refine o System Prompt (especialmente o campo Contexto atual) e rode o prompt do Passo 06 de novo. Os arquivos continuam carregados — não precisa subir de novo.

Desafio
10. Implementando na sua empresa — ritual semanal
Escale para 10 notas reais + estabeleça o ritual de processamento semanal

Agora que você tem o sistema funcionando com 3 notas, leve para escala real. Use as 10 notas do Desafio da AP02 e estabeleça um ritual semanal de processamento.

  1. Volte ao Claude Project. Abra o Project "Segundo Cérebro — [Seu nome]" criado na AP.
  2. Atualize os arquivos de contexto. Remova os 3 arquivos da AP e suba as 10 notas reais do Desafio da AP02 (que você preencheu com seu conteúdo de founder). Mantenha o System Prompt — ele continua válido.
  3. Refine o System Prompt se necessário. Agora que o Claude tem mais notas, vale revisitar o campo Contexto atual e enriquecer com 2-3 prioridades estratégicas que você quer que ele tenha em mente sempre.
  4. Gere 4 outputs diferentes esta semana. Use o Project para 4 finalidades distintas:
    • Segunda: plano de ação dos próximos 30 dias
    • Quarta: briefing executivo de uma decisão específica para o time
    • Sexta: insight original para post de autoridade
    • Domingo: retrospectiva da semana (o que aconteceu nas notas?)
  5. Estabeleça o ritual semanal. Bloqueie 30 min no calendário toda sexta-feira para: (a) capturar 5-10 novas notas da semana no Obsidian, (b) atualizar os arquivos no Claude Project, (c) pedir ao Claude uma análise da nova rede. Em 4 semanas você terá ~40 notas e um Claude que conhece profundamente seu pensamento.

Parâmetros do desafio

Tempo do setup inicial30 a 45 minutos
Tempo semanal recorrente30 minutos por semana
💡 Critério de qualidade — semana 4

Se em 4 semanas você consegue abrir o Claude Project e fazer uma pergunta sobre uma decisão estratégica e ele responde citando 3 ou mais notas suas com vocabulário próprio, o sistema funcionou. Se as respostas continuam genéricas, a rede do vault precisa de mais notas ou mais links — não é problema do Claude.
