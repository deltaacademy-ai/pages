Introdução
02. Os 3 elementos do fluxo de publicação
O que precisa estar pronto antes de rodar a skill

Publicar uma aula da Delta envolve três elementos que precisam estar prontos antes de você rodar a skill. Conhecer cada um evita travas no meio do caminho.

Elementos do fluxo

  • Terminal + Git — você usa o terminal do macOS para clonar o repositório oficial deltaacademy-ai/pages na sua máquina. Esse é o diretório onde a skill vai gravar os arquivos da sua aula.
  • Claude Code apontado para a pasta pages — o Claude Code precisa estar com o workspace aberto exatamente na pasta clonada. Sem isso, a skill /criar-nova-aula-delta não enxerga onde gravar.
  • Skill /criar-nova-aula-delta + roteiro .md — a skill da organização Delta Academy automatiza a geração da página HTML, CSS, sidebar e deploy. Ela precisa receber o roteiro .md da aula para gerar o conteúdo.
Introdução
03. Skill como conhecimento empacotado
Por que existe a skill /criar-nova-aula-delta

A Delta Academy tem dezenas de aulas no ar, todas com a mesma identidade visual, a mesma navegação, a mesma estrutura de pílulas, prints, blocos de copiar e insight-boxes. Se cada professor precisasse construir tudo isso do zero a cada aula nova, seriam horas de CSS, HTML e configuração — antes mesmo de pensar no conteúdo pedagógico.

A skill criar-nova-aula-delta resolve isso de uma vez. Ela é o conhecimento da Delta Academy empacotado em uma instrução executável: paleta de cores, fontes, componentes, layout, deploy no GitHub Pages — tudo encapsulado. Você só descreve o conteúdo da aula no .md, e a skill aplica todo o padrão Delta automaticamente.

Dica

Pense numa skill como uma receita de bolo que já vem com o forno embutido. Você fornece os ingredientes (o roteiro .md da aula), e a skill cuida do resto — temperatura, tempo, decoração, entrega. Toda aula que sai por essa skill vem com o mesmo padrão visual da Delta, sem você precisar se preocupar com isso.

A vantagem para o programa é a consistência: 100 aulas criadas por 30 professores diferentes ficam visualmente idênticas, porque todas passaram pela mesma skill. A vantagem para você como professor é o foco: você gasta sua energia no que importa — o conteúdo pedagógico — e não em decisões de design.

04. Sua atividade prática
20 minutos · 8 passos sequenciais

Modelo recomendado
Sonnet 4.6

Padrão do Claude Code, dá conta com folga da skill. Como roda localmente, não consome créditos do plano gratuito da web.

Você vai executar os 8 passos do fluxo oficial de publicação de uma aula da Delta Academy — do clone do repositório à verificação da página publicada no GitHub Pages.

Dica

Esse fluxo é o mesmo que você vai repetir toda vez que criar uma aula nova. Faça uma vez agora com atenção — depois vira automático. A maior parte do tempo é navegação de interface; o trabalho pesado (gerar HTML, CSS, sidebar, deploy) fica todo com a skill.

Importante

Em nenhum momento desta aula a skill, o GitHub ou o Claude Code vai pedir que você crie ou forneça uma senha. Se em algum ponto o terminal sugerir criar uma senha, negue/ignore. Toda autenticação necessária acontece via a sua conta já logada no Claude Code e no GitHub.

Atividade Prática
05. Passo a passo
8 passos sequenciais — do clone à verificação
PASSO 01Clonar o repositório do GitHub da Delta Academy

Abra o Terminal do macOS e digite o comando abaixo para clonar o repositório oficial de aulas da Delta Academy na sua máquina. Esse comando cria uma pasta chamada pages no diretório onde você executou o comando (geralmente o seu diretório home /Users/{seu-usuario}/).

Comando para clonar o repositório
git clone https://github.com/deltaacademy-ai/pages
PASSO 02Verificar a pasta pages no seu computador

Após o clone terminar, abra o Finder e verifique que existe uma pasta chamada pages dentro do seu usuário (no exemplo, dentro de rafael-arevalo). Essa pasta é o repositório local que vai receber os arquivos da sua aula.

PASSO 03Abrir a pasta pages no Claude Code

Abra o Claude Code e use a opção "Abrir pasta..." para selecionar a pasta pages criada no passo anterior. O Claude Code vai carregar o workspace apontando para essa pasta.

PASSO 04Chamar a skill /criar-nova-aula-delta

No campo de prompt do Claude Code, chame a skill digitando o slash command abaixo. A partir daqui, a skill vai começar a fazer perguntas no terminal — responda cada uma à medida que aparecerem.

Slash command da skill
/criar-nova-aula-delta
PASSO 05Fornecer seu username do GitHub

A skill vai perguntar qual é o seu username do GitHub. Digite o seu username (o mesmo que você usa no github.com) e envie.

Importante

Se em algum momento o terminal pedir para você criar ou fornecer uma senha, recuse/ignore. A skill não precisa de senha nenhuma — autenticação já está coberta pela sua conta logada no Claude Code e no GitHub.

PASSO 06Selecionar a opção "A" (módulo novo com aulas)

A skill vai mostrar uma lista de opções de fluxo. Selecione a opção "A — Módulo novo com aulas" para criar um módulo novo do zero. Em seguida, adicione o arquivo .md com o roteiro da aula quando a skill solicitar.

PASSO 07Confirmar criação do módulo novo

Caso a skill peça uma confirmação adicional, confirme que você quer criar um novo módulo de conteúdo Delta. A partir daqui a skill assume o controle: ela vai criar a pasta do módulo, gerar o HTML a partir do seu .md, configurar a sidebar, fazer commit e push para o GitHub.

Importante

A partir desse passo, não interrompa a execução da skill. Ela vai rodar vários comandos em sequência (criar arquivos, fazer commit, fazer push). Se cancelar no meio, pode deixar o repositório em estado inconsistente. Aguarde até a skill mostrar a URL da aula publicada.

PASSO 08Verificar a página HTML publicada

Quando a skill terminar, ela vai fornecer a URL pública da aula publicada no GitHub Pages. Copie essa URL, cole no navegador e confirme que a página carrega com a identidade visual Delta aplicada. Em seguida, navegue até o item Passo a passo dentro da seção ATIVIDADE PRÁTICA da aula para verificar que o conteúdo do seu roteiro .md foi renderizado corretamente.

Dica

Se algo visualmente estiver fora do padrão Delta, ou se a página retornar 404, o cache do GitHub Pages pode estar servindo uma versão antiga ou o deploy ainda está rodando. Espere 1–2 minutos e faça um hard refresh (Cmd + Shift + R no Chrome/Firefox, Cmd + Option + R no Safari).

Atividade Prática
06. O que você tem ao final desta atividade
Os 5 entregáveis garantidos

Você sai desta atividade com

  • Um módulo novo no repositório deltaacademy-ai/pages com pasta criada, arquivos commitados e push feito para a branch principal.
  • Pelo menos uma aula HTML publicada no GitHub Pages acessível por URL pública, com a identidade visual Delta aplicada (sidebar, breadcrumb, paleta, componentes).
  • Seu username do GitHub registrado na skill — reutilizado em todas as próximas aulas que você publicar.
  • O fluxo de 8 passos rodado de ponta a ponta uma vez — da próxima vez que criar uma aula, esse fluxo vai estar muito mais rápido porque já é familiar.
  • A prova viva de que a skill criar-nova-aula-delta funciona no seu setup — workspace configurado, autenticação OK, deploy automatizado funcionando.
Insight

Você acabou de publicar sua primeira aula da Delta Academy. A partir daqui, toda vez que tiver um roteiro .md de aula pronto, basta repetir esse fluxo de 8 passos — e em menos de 20 minutos você tem uma aula nova no ar. A skill faz o trabalho pesado; você foca no conteúdo pedagógico.
