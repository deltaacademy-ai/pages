01. Onde você vai chegar
O entregável da atividade prática

Ao final desta atividade, você terá publicado seu primeiro módulo novo no repositório oficial da Delta Academy, contendo pelo menos uma aula HTML viva e acessível por URL no GitHub Pages. Esse é o entregável de cada professor recém-chegado: sair desta aula com sua primeira página publicada e funcionando.

Introdução
02. Os 3 elementos do fluxo de publicação
O que precisa estar pronto antes de rodar a skill

Publicar uma aula da Delta envolve três elementos que precisam estar prontos antes de você rodar a skill. Conhecer cada um evita travas no meio do caminho.

Elementos do fluxo

  • Repositório clonado localmente — a pasta pages do deltaacademy-ai precisa estar no seu computador, dentro do /Users/seunome/ (ou caminho equivalente). Esse é o diretório onde a skill vai gravar os arquivos da sua aula.
  • Claude Code aberto na pasta certa — o Claude Code precisa estar com o workspace apontando para a pasta pages que você clonou. Sem isso, a skill /criar-nova-aula-Delta não enxerga onde gravar.
  • Skill /criar-nova-aula-Delta disponível — é a skill da organização Delta Academy que automatiza toda a geração da página HTML, do CSS, da sidebar e do deploy para o GitHub Pages.
Introdução
03. Skill como conhecimento empacotado
Por que existe a skill /criar-nova-aula-Delta

A Delta Academy tem dezenas de aulas no ar, todas com a mesma identidade visual, a mesma navegação, a mesma estrutura de pílulas, prints, blocos de copiar e insight-boxes. Se cada professor precisasse construir tudo isso do zero a cada aula nova, seriam horas de CSS, HTML e configuração — antes mesmo de pensar no conteúdo pedagógico.

A skill criar-nova-aula-Delta resolve isso de uma vez. Ela é o conhecimento da Delta Academy empacotado em uma instrução executável: paleta de cores, fontes, componentes, layout, deploy no GitHub Pages — tudo encapsulado. Você só descreve o conteúdo da aula, e a skill aplica todo o padrão Delta automaticamente.

Dica

Pense numa skill como uma receita de bolo que já vem com o forno embutido. Você fornece os ingredientes (o conteúdo da sua aula), e a skill cuida do resto — temperatura, tempo, decoração final, entrega. Toda aula que sai por essa skill vem com o mesmo padrão visual da Delta, sem você precisar se preocupar com isso.

A vantagem para o programa é a consistência: 100 aulas criadas por 30 professores diferentes ficam visualmente idênticas, porque todas passaram pela mesma skill. A vantagem para você como professor é o foco: você gasta sua energia no que importa — o conteúdo pedagógico — e não em decisões de design.

Padrão do Claude Code, dá conta com folga da skill. Como roda localmente, não consome créditos do plano gratuito da web.

Você vai executar os 16 passos do fluxo oficial de publicação de uma aula da Delta Academy. O fluxo está dividido em três fases sequenciais:

As 3 fases do fluxo

  • Fase 1 — Configuração do workspace · abrir o Finder, navegar até a pasta pages e apontar o Claude Code para ela (Passos 01 a 08).
  • Fase 2 — Execução da skill · rodar o slash command /criar-nova-aula-Delta, configurar seu username de professor e escolher a opção de módulo novo (Passos 09 a 12).
  • Fase 3 — Verificação no GitHub Pages · copiar a URL gerada, abrir no navegador e validar que a aula está publicada e visualmente correta (Passos 13 a 16).
Dica

Esse fluxo de 16 passos é o mesmo que você vai repetir toda vez que criar uma aula nova. Faça uma vez agora com atenção — depois vira automático. A maior parte do tempo nesse fluxo é navegação de interface; o trabalho pesado (gerar HTML, CSS, sidebar, deploy) fica todo com a skill.

Atividade Prática
05. Passo a passo
16 passos separados por fase
Fase 1 Configuração do workspace Passos 01–08
PASSO 01Abrir o Finder a partir do Dock

Clique no ícone do Finder no Dock do macOS para abri-lo. O Finder vai abrir na janela padrão configurada no seu sistema — pode ser Recentes, Documentos ou outro local. Não importa onde abriu; nos próximos passos você vai navegar para o lugar certo.

PASSO 02Navegar até o diretório home

No Finder aberto, vá até o diretório home do seu usuário (a pasta com o ícone de casinha, geralmente chamada com o seu nome de usuário). Você pode usar o atalho Cmd + Shift + H ou clicar na pasta do seu nome na barra lateral do Finder.

PASSO 03Localizar a pasta pages

Dentro do diretório home, localize a pasta chamada pages. Essa é a pasta do repositório deltaacademy-ai/pages que você clonou como pré-requisito desta aula. Se você não encontrar essa pasta, é sinal de que o repositório ainda não foi clonado — pare aqui e clone antes de continuar.

Importante

Se a pasta pages não existir no seu home, abra o Terminal e rode git clone git@github.com:deltaacademy-ai/pages.git no diretório home antes de seguir. Sem essa pasta clonada, a skill não tem onde gravar a aula.

PASSO 04Selecionar a pasta pages

Clique uma vez sobre a pasta pages para selecioná-la (não entre dentro dela — só selecionar é suficiente). A pasta deve aparecer destacada/azulada indicando que está selecionada. Mantenha o Finder aberto nesse estado — você vai precisar dele aberto para o próximo passo.

PASSO 05No Claude Code, clicar em "Abrir pasta..."

Agora abra o Claude Code (se ele já não estiver aberto). Na interface inicial do Claude Code, clique em "Abrir pasta..." (ou "Open folder..." dependendo do idioma da interface). Isso vai abrir um diálogo de seleção de pasta nativo do macOS.

PASSO 06Selecionar a pasta do repositório no diálogo

No diálogo de seleção que abriu, navegue até o diretório home e selecione a pasta pages (ou a pasta exata em que você executou o git clone se foi em outro lugar). Confirme que a pasta selecionada é a do repositório deltaacademy-ai/pages.

PASSO 07Confirmar a abertura clicando em "Abrir"

No canto inferior direito do diálogo de seleção, clique no botão "Abrir" para confirmar a escolha da pasta. O Claude Code vai começar a carregar o workspace apontando para essa pasta.

PASSO 08Confiar no workspace

Na primeira vez que você abre uma pasta no Claude Code, ele pede confirmação de segurança. Uma janela vai aparecer perguntando se você confia no conteúdo dessa pasta. Clique em "Confiar no Workspace" (ou "Trust Workspace") para autorizar o Claude Code a ler e executar arquivos dentro da pasta.

Importante

Esse aviso só aparece na primeira vez. Nas próximas vezes que você abrir a pasta pages, o Claude Code já vai considerá-la confiável e pular esse passo.

Fase 2 Execução da skill Passos 09–12
PASSO 09Digitar o comando /criar-nova-aula-Delta

Com o workspace já aberto e confiável, vá até o campo de prompt do Claude Code (a caixa de texto onde você normalmente conversa com o Claude). Digite o slash command exato e aperte Enter. O Claude Code vai detectar que é um comando de skill da organização Delta Academy e iniciar o fluxo de criação.

Slash command
/criar-nova-aula-Delta
PASSO 10Inserir seu username de professor

A skill vai perguntar qual é o seu username de professor da Delta Academy. Se você já é professor cadastrado, insira o username que você já usa. Se essa é sua primeira aula, invente um username novo (algo curto, em lowercase, sem espaços — ex: rafa, mariag, pedrocosta). Esse username vai te identificar como autor das aulas que criar.

Dica

Anote seu username em algum lugar fácil de acessar. Você vai precisar dele toda vez que criar uma aula nova — e ele precisa ser exatamente o mesmo para a skill reconhecer suas aulas anteriores.

PASSO 11Selecionar "A — Módulo novo com aulas"

A skill vai te mostrar uma lista de opções de fluxo. Como esta é sua primeira aula e você quer criar um módulo novo do zero com pelo menos uma aula dentro, selecione a opção "A — Módulo novo com aulas". Não selecione as opções de "adicionar aula a módulo existente" — essas você vai usar quando voltar para complementar um módulo já criado.

PASSO 12Confirmar a criação do módulo digitando "A"

A skill vai pedir uma confirmação adicional antes de começar a gerar os arquivos. Digite a letra "A" no campo de prompt e envie. Isso confirma a sua escolha e libera a skill para começar a gerar o módulo novo com a aula dentro. A partir daqui a skill assume o controle: ela vai criar a pasta do módulo, gerar o HTML, o CSS, configurar a sidebar e preparar o commit.

Importante

A partir desse passo, não interrompa a execução da skill. Ela vai rodar vários comandos em sequência (criar arquivos, fazer commit, fazer push para o GitHub). Se você cancelar no meio, pode deixar o repositório em estado inconsistente. Aguarde até a skill terminar e mostrar a URL da aula publicada.

Fase 3 Verificação no GitHub Pages Passos 13–16
PASSO 13Copiar a URL da aula publicada

Quando a skill terminar a execução, ela vai exibir no terminal a URL pública da aula que acabou de ser publicada no GitHub Pages. A URL tem o formato https://tutoriais.deltaacademy.ai/{seu-modulo}/{sua-aula}. Selecione e copie essa URL (Cmd+C) — você vai colá-la no navegador no próximo passo.

PASSO 14Colar a URL no navegador e abrir

Abra seu navegador padrão (Chrome, Safari, Firefox), clique na barra de endereço, cole a URL (Cmd+V) e aperte Enter. O GitHub Pages pode levar de 30 segundos a 2 minutos para terminar de fazer o deploy nas primeiras vezes — se a página não abrir de cara, espere um pouco e atualize.

Importante

Se a página exibir erro 404 mesmo depois de 2 minutos, abra o GitHub do repositório deltaacademy-ai/pages e verifique se o último commit (feito pela skill) está listado nos commits. Se estiver, o deploy ainda está rodando — espere mais 1 minuto e atualize. Se o commit não aparecer, algo travou no push e você precisa rerodar a skill.

PASSO 15Clicar no link da aula recém-criada

A URL pode te levar para a página índice do módulo que você criou (não direto para a aula). Nesse caso, você vai ver uma lista com o título da aula que a skill gerou — algo como "Benchmark da Vaga" ou similar dependendo do conteúdo. Clique nesse link para abrir a página da aula em si.

PASSO 16Verificar a aula publicada

A aula deve carregar com a identidade visual completa da Delta Academy: sidebar à esquerda com a navegação dos itens, breadcrumb no topo, banner do entregável na introdução, paleta de cores Delta Board (verde escuro, dourado, cream). Navegue pelas seções da aula clicando nos itens da sidebar para confirmar que tudo está funcionando.

Dica

Se algo visualmente estiver fora do padrão Delta (cor errada, sidebar quebrada, fonte diferente), provavelmente o cache do GitHub Pages ainda está servindo uma versão antiga. Faça um hard refresh no navegador (Cmd + Shift + R no Chrome/Firefox, Cmd + Option + R no Safari) e o problema costuma resolver.

Atividade Prática
06. O que você tem ao final desta atividade
Os 5 entregáveis garantidos

Você sai desta atividade com

  • Um username de professor da Delta Academy criado e registrado — vai ser reutilizado em todas as próximas aulas que você publicar.
  • Um módulo novo no repositório deltaacademy-ai/pages com pasta criada, arquivos commitados e push feito para a branch principal.
  • Pelo menos uma aula HTML publicada no GitHub Pages acessível por URL pública, com a identidade visual Delta aplicada (sidebar, breadcrumb, paleta, componentes).
  • O fluxo de 16 passos rodado de ponta a ponta uma vez — da próxima vez que você criar uma aula, esse fluxo vai estar muito mais rápido porque já é familiar.
  • A prova viva de que a skill criar-nova-aula-Delta funciona no seu setup — workspace configurado, autenticação OK, deploy automatizado funcionando.
Insight

Você acabou de publicar sua primeira aula da Delta Academy. A partir daqui, toda vez que tiver um rascunho de aula pronto, basta repetir esse fluxo de 16 passos — e em menos de 30 minutos você tem uma aula nova no ar. A skill faz o trabalho pesado; você foca no conteúdo pedagógico.
