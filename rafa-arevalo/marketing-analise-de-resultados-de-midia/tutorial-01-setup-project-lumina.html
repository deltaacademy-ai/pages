AP01 — Setup do Project “Resultados de Mídia — Lumina”

Entregável final: um Claude Project configurado e pronto para receber as 3 Skills das próximas APs — o ambiente onde a Nathália Raoli (Head de Marketing da Lumina) vai operar toda segunda-feira de manhã para fechar a análise da semana.

Introdução
01. Objetivo de aprendizagem
Por que configurar um Project antes de qualquer análise

Configurar um Claude Project que carregue permanentemente o contexto da operação de mídia da Lumina — budget, metas, regras de atribuição, identidade visual — para que toda análise futura comece já briefada, sem precisar reexplicar nada toda segunda-feira.

O que isso destrava

  • Análises que comparam com a meta real (ROAS 3.5x pixel, 2.0x CRM) em vez de números genéricos
  • Outputs no formato Lumina (tabela + 1 frase de “o que isso significa”) por padrão
  • Linguagem executiva, sem explicar termos básicos para um profissional de marketing
  • Atribuição first-click via CRM sempre separada do que o pixel reporta — sem confusão de fonte
Introdução
02. Project como ambiente que carrega contexto permanente
A diferença entre freelancer e analista interno

Conversar com o Claude em um chat solto é como contratar um analista freelancer toda segunda-feira: o profissional é competente, mas começa do zero. Você precisa explicar o que é a Lumina, quais são os canais, quanto é o budget, qual a meta de ROAS, qual a identidade visual da marca. A cada nova conversa, o mesmo briefing.

Um Project resolve isso de forma permanente. Você configura o contexto uma vez — descrição da empresa, regras de atribuição, identidade visual, arquivos de referência — e o Claude carrega tudo automaticamente em cada nova conversa dentro daquele Project. É a diferença entre freelancer e analista interno: o segundo já chega sabendo onde o cafezinho fica.

Chat solto, sem Project
“Analise este export do Meta Ads da semana 12-18 mai”
Resposta genérica. Claude calcula CTR e CPM, mas não sabe qual o budget alvo, não conhece a meta de ROAS, não sabe se o resultado é bom ou ruim para a Lumina. Faltam todos os ganchos contextuais.
Mesma planilha dentro do Project
Mesmo prompt curto, agora dentro do Project Lumina
Resposta executiva. Claude compara com o budget mensal de R$ 164k, sinaliza se bateu a meta de ROAS, menciona o lançamento de whey daquela semana, e devolve já no formato padrão da Lumina. Pronto pra reunião.

04. Prompt comparativo da demonstração
O mesmo prompt, rodado em duas conversas separadas

O mesmo prompt curto é rodado em duas conversas separadas — uma sem Project, outra dentro do Project Lumina já configurado. A única variável é o contexto carregado.

Prompt da demonstração
Analise este export do Meta Ads da Lumina referente à semana de 12 a 18 de maio de 2026. Resuma o que aconteceu na semana e diga se a performance foi boa.
Dica do professor

Rode o prompt no chat solto primeiro — o output curto cabe na tela e dá tempo de comparar com o output do Project lado a lado. Quando entrar no Project, mantenha a janela do chat solto visível para os alunos verem a diferença em tempo real.

Demonstração
05. O que observar durante a demonstração
6 padrões visíveis ao comparar os dois outputs
  • Sem Project, sem meta. O Claude diz “ROAS de 3.8x é geralmente considerado bom”, sem referência da Lumina.
  • Sem Project, explica termos básicos. CTR, CPM, ROAS — tudo é explicado, como se o usuário fosse leigo.
  • Com Project, compara com a meta mensal. ROAS 3.5x pixel é o piso aceito — aparece automaticamente.
  • Com Project, menciona o lançamento de whey. O evento de maio está carregado no plano de mídia.
  • Com Project, devolve no formato padrão Lumina. Tabela + 1 frase de “o que isso significa”.
  • A diferença não está no prompt. Está no contexto que foi configurado uma vez e está disponível para sempre.
Atividade Prática
06. Sua atividade prática
12 minutos · setup puro via interface · zero mensagens consumidas

Você vai configurar o Project “Resultados de Mídia — Lumina” copiando 3 blocos de texto prontos e fazendo upload de 2 arquivos de contexto. Esta AP não envia nenhuma mensagem ao Claude — é puro setup via interface.

Ficha da atividade

Duração estimada12 minutos
Modelo recomendadoClaude Sonnet 4.6
Plano necessárioGratuito (Sonnet 4.6 é o modelo padrão do plano free)
Mensagens consumidasZero — apenas preenchimento de campos e upload
Por que esta AP não consome mensagens

O plano gratuito do Claude permite cerca de 5 mensagens a cada 5 horas. Esta aula foi calibrada para caber exatamente nesse limite: as 5 mensagens ficam reservadas para as AP02 a AP06, onde o Claude processa dados e gera outputs. A AP01 é apenas configuração — você preenche campos e faz upload, sem disparar processamento.

Atividade Prática
07. Passo a passo
7 passos — copiar 3 blocos, baixar e subir 2 arquivos
1
Acesse a área de Projects do Claude. (30 seg)
Abra claude.ai, faça login e clique em Projetos no menu lateral. Depois clique em + Novo projeto.
2
Cole o Nome do Project no campo “No que você está trabalhando?”. (20 seg)
O bloco para copiar está logo abaixo deste passo a passo, identificado como Bloco A — Nome.
3
Cole a Descrição do Project no campo “O que você quer realizar?”. (30 seg)
Use o Bloco B — Descrição. Depois clique em Criar projeto.
4
Abra o editor de Instruções. (30 seg)
Na tela do Project, localize a seção Instruções e clique em Editar instruções.
5
Cole as Instruções do Project — é o System Prompt. (1 min)
Use o Bloco C — Instruções (mais longo, ~600 palavras). Cole tudo e clique em Salvar instruções.
6
Baixe os 2 arquivos de contexto. (30 seg)
Use os botões de download mais abaixo, na seção Arquivos de contexto para download. Os 2 arquivos vão para sua pasta de Downloads.
7
Faça upload dos 2 arquivos no Project. (1 min)
Volte ao Project, abra a seção Arquivos, clique em + → Carregar do aparelho e selecione os 2 arquivos. Confirme que aparecem listados.
Importante — System Prompt

O conteúdo das Instruções (Bloco C) é o System Prompt do Project — ele define a identidade e as regras que o Claude vai seguir em todas as conversas deste Project. É a peça mais valiosa do setup.

Blocos para copiar

Bloco A — Nome do Project
Resultados de Mídia — Lumina
Bloco B — Descrição do Project
Project para análise semanal de performance de mídia paga da Lumina. Toda segunda-feira de manhã consolida Meta Ads + Google Ads + GA4 + CRM da semana anterior, apura ROAS real via atribuição first-click e gera painel HTML executivo + comentário para o CMO.
Bloco C — Instruções (System Prompt)
Você é o analista de performance de mídia sênior dedicado à Lumina — marca de wellness premium que vende balas de goma funcionais (pré-treino, creatina, whey protein) via e-commerce próprio e 2 lojas físicas (uma em São Paulo, no Jardins; outra no Rio, no Leblon). A Lumina investe aproximadamente R$ 164.000 por mês em mídia paga, dividido entre Meta Ads (~R$ 108.000) e Google Ads (~R$ 56.000). # Tom e identidade Tom direto, executivo, sem floreio. Orientado a ação ("o que fazer com este número") e não apenas a relatar números. Você não explica termos básicos como CTR, CPM ou ROAS — assume que o interlocutor é um profissional de marketing. # Atribuição (regra crítica) A Lumina usa atribuição **first-click** como padrão contábil oficial. Sempre que você falar de "ROAS real" ou "receita real", refere-se à receita do CRM atribuída ao canal da primeira visita do cliente. Sempre que falar de "ROAS reportado" ou "receita pixel", refere-se ao valor de conversão reportado pelo Meta Ads ou Google Ads. **Nunca misture os dois sem explicitar a fonte.** # Diretrizes de análise Priorize 4 métricas: gasto, receita real (CRM), ROAS real (CRM), CPA real (CRM). Sempre que possível compare com a semana anterior. Destaque variações acima de 10% como "atenção" e acima de 25% como "alerta crítico". Sempre conecte número a ação — não devolva apenas relatório sem recomendação. # Formato padrão de output Tabelas em markdown para dados quantitativos. Prosa estruturada para análise narrativa. Após cada tabela, inclua 1 frase de "o que isso significa". Nunca devolva apenas números sem narrativa. Valores monetários sempre formatados como `R$ 1.234,56`. ROAS no formato `0.00x` (ex: 2.14x). Percentuais com 1 casa decimal. # Stack de Skills Este Project usa 3 Skills sequenciais para o ciclo semanal (Skills adicionadas na conta Claude do usuário, disponíveis em qualquer Project): 1. consolidar-midia-lumina — recebe os 3 CSVs (Meta, Google, GA4) e gera a consolidação executiva 2. extrair-receita-crm-lumina — recebe o CSV do CRM e apura a receita real por canal 3. calcular-roas-real-lumina — cruza as duas anteriores e produz o reconciliation report final Depois das 3 Skills, o Project gera o painel HTML via Claude Design e configura automação semanal via Cowork. # Identidade visual nos outputs Quando gerar peças visuais (painel HTML, relatórios formatados), siga o brand book carregado no Project: - Paleta: verde-musgo #5C7A5A (primária), coral terroso #C97A5A (destaque), branco quente #FBF9F5 (fundo) - Tipografia: Fraunces para títulos, Inter para corpo - Estética: minimalismo wellness premium — muito espaço negativo, linhas finas, dados como protagonistas
