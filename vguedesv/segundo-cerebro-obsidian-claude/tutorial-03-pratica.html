Aula 03 — Prática: construa o seu, do zero

Mão na massa. Ao final desta aula você terá um segundo cérebro funcionando: estrutura criada, primeira fonte ingerida, primeira pergunta respondida e tudo sincronizado no GitHub. Sem precisar ser programador.

01. O que vamos fazer
O mapa dos próximos 80 minutos

Roteiro

  • Instalar as ferramentas (Obsidian + Claude Code)
  • Criar a pasta da vault e abrir o Claude Code dentro dela
  • Colar o prompt de bootstrap — o Claude monta tudo sozinho
  • Ingerir a primeira fonte e fazer a primeira pergunta
  • Conectar ao GitHub e sincronizar
A grande virada de método

Você não vai criar pastas e escrever o CLAUDE.md na mão. Vai colar um único prompt e o Claude Code constrói a estrutura inteira. Menos erro, mais resultado.

Atividade Prática
02. Passo a passo
Do zero ao primeiro cérebro funcionando
PASSO 01Instale o Obsidian

Acesse obsidian.md, baixe para seu sistema e abra. Clique em "Create new vault", dê um nome (ex: meu-cerebro), escolha uma pasta e confirme.

Detalhamento: anote ONDE você salvou a vault — vamos precisar do caminho no PASSO 03.

PASSO 02Instale o Claude Code

Abra o terminal (Mac: Cmd+Espaço → "Terminal"; Windows: menu Iniciar → "Terminal"). Instale o Claude Code seguindo o site oficial e faça login digitando claude.

Detalhamento: precisa de assinatura Claude (Pro/Max) ou chave de API. Se aparecer command not found: npm, instale antes o Node.js em nodejs.org (versão LTS).

PASSO 03Abra o Claude Code dentro da vault

No terminal, entre na pasta da vault com cd e abra o Claude Code com claude. Tudo que ele criar vai dentro desta pasta.

Detalhamento: truque para o caminho — digite cd com um espaço e arraste a pasta do Finder/Explorer para o terminal; o caminho cola sozinho. Depois dê Enter e rode claude.

Terminal
cd ~/Documents/meu-cerebro claude
PASSO 04Cole o prompt de bootstrap

Copie o prompt abaixo inteiro, cole no Claude Code (aberto na pasta da vault) e dê Enter. Ele cria as pastas, o CLAUDE.md, os comandos /ingest /query /lint, o index.md, o log.md, os templates e a taxonomy.

Prompt de bootstrap — cole no Claude Code
Quero que você construa, nesta pasta, a estrutura completa de um "segundo cérebro" — um wiki pessoal mantido por você (LLM), no padrão LLM Wiki do Andrej Karpathy, com melhorias da comunidade. Esta pasta é uma vault do Obsidian. PRINCÍPIOS: - raw/ guarda fontes brutas: você LÊ mas NUNCA edita. - wiki/ guarda as páginas que você escreve e mantém: você é dono total. - Conecte tudo com [[wikilinks]] do Obsidian. Cross-references densas. - Marque sempre o que é fato vs. inferência (provenance). - Títulos LEGÍVEIS: o nome de cada página é uma frase curta e humana — sem IDs, números longos ou prefixos técnicos. - Não invente: se a fonte não diz, não vai pro wiki. - Português. Páginas curtas e densas. Listas melhor que parágrafos. CRIE ESTA ESTRUTURA DE PASTAS: raw/artigos/ raw/notas/ raw/transcricoes/ wiki/conceitos/ wiki/entidades/ wiki/fontes/ _meta/templates/ CRIE O ARQUIVO CLAUDE.md (na raiz) com as regras das três operações: INGEST (/ingest <arquivo>): 1. Leia a fonte completa. 2. QUALITY GATE: vale a pena? Se for ruído ou redundante, me avise antes de ingerir. 3. Me diga os 3-5 takeaways principais numa mensagem curta, ANTES de escrever. 4. Crie uma página-resumo em wiki/fontes/<slug>.md. 5. Extraia conceitos e entidades: crie página própria só se aparecer em 2+ fontes OU for central a esta. Menção de passagem = só link inline. Se a fonte nova contradiz uma página, marque a contradição. 6. ANTES de ligar ao index, posicione a página em HUB(s) intermediários: a. DESCUBRA o que já existe (ls wiki/projetos/, ls wiki/entidades/) e REUTILIZE um hub/lugar já criado — nunca duplique um cluster. b. FACETAS (vários pais): assunto (hub em projetos/), lugar (entidade, ex: entidades/recife), pessoa/empresa (entidade). Ex: café em Recife → [[projetos/gastronomia]] + [[entidades/recife]]. c. HIERARQUIA: hubs aninham (Restaurantes→Gastronomia; Recife→Nordeste). d. Crie um intermediário novo se a categoria for natural/reutilizável (uma cidade, um assunto) — mesmo com 1 folha. Só deixe direto no index o que realmente não tem categoria. NÃO deixe o index virar estrela achatada. e. Backlink (## Faz parte de) + tag tema/<slug> em cada página do cluster (habilita cor no grafo: Graph view → Groups, busca tag:#tema/<slug>). 7. Atualize index.md (apontando os HUBS, não cada folha de um cluster) e log.md. 8. Me resuma o que mudou. Uma ingestão típica toca 5-15 páginas. QUERY (/query <pergunta>) — tiered retrieval, comece barato: 1. Leia index.md e identifique candidatas pelo título + summary. 2. Leia o frontmatter (summary, tags, sources) das candidatas. 3. Só então abra o corpo das que importam. 4. Cite SEMPRE: toda afirmação linka [[a página]] que a sustenta. 5. Se inferiu além do escrito, diga: "minha leitura, não está escrito". 6. Se a resposta foi síntese não-trivial, pergunte se quero salvá-la como página. LINT (/lint) — reporte, não conserte sem perguntar: páginas órfãs, [[wikilinks]] quebrados, conceitos citados em 3+ páginas sem página própria, contradições não marcadas, frontmatter faltando, tags fora da taxonomy, e grupos de ≥3 folhas do mesmo tema penduradas direto no index (que deveriam ter um hub em projetos/). Ao final, pergunte quais consertar. FRONTMATTER OBRIGATÓRIO em toda página de wiki/: --- type: conceito | entidade | fonte summary: "uma frase de 1 linha" tags: [tag1, tag2] provenance: extracted | inferred | ambiguous sources: ["[[fontes/slug]]"] created: AAAA-MM-DD updated: AAAA-MM-DD --- provenance: extracted = fato direto; inferred = síntese sua; ambiguous = fontes discordam. CONVENÇÃO DE TÍTULOS (para o grafo do Obsidian ficar legível): - O nome de cada página é uma frase curta e humana — ex: "Estoicismo - dicotomia do controle", NUNCA "video-7635764687237401874" ou "fonte-tiktok-123". - Nada de IDs, números longos, URLs ou prefixos técnicos no nome do arquivo. - Em fontes (vídeos/artigos), o título descreve o CONTEÚDO ("Rota barata para a Ásia"), não a origem. IDs, URL e data ficam só no frontmatter. - Nome do arquivo = kebab-case do título legível (ex: estoicismo-dicotomia-do-controle.md). LOG (log.md) — toda operação vira entrada parseável: ## [AAAA-MM-DD] ingest | título da fonte - páginas tocadas: [[a]], [[b]] - novas: [[c]] - takeaway: uma linha INDEX (index.md) — catálogo organizado por seções (Conceitos, Entidades, Fontes). Cada linha: [[link]] — summary de uma linha. Atualize a cada ingestão. ESTILO: Português. Páginas curtas e densas. Listas > parágrafos. CO-EVOLUÇÃO: quando uma convenção atrapalhar, proponha mudança no CLAUDE.md. CRIE OS SLASH COMMANDS em .claude/commands/ (ingest.md, query.md, lint.md), cada um chamando o protocolo do CLAUDE.md e usando $ARGUMENTS onde recebe input. CRIE OS TEMPLATES em _meta/templates/ (conceito.md, entidade.md, fonte.md) com o frontmatter acima e seções de corpo apropriadas. CRIE _meta/taxonomy.md com tags iniciais (lowercase, sem espaços): domínios (tech, ai, negocios, pessoal, aprendizado), tipos (pessoa, empresa, produto, ferramenta, framework, artigo, video, podcast, nota), status (stub, draft, core, revisar) e temas de cluster (tag tema/<slug>, ex: tema/viagens — um por hub). CRIE index.md inicial com seções vazias e log.md com uma entrada de bootstrap. Ao terminar, me mostre a estrutura criada e me explique como ingerir minha primeira fonte.
Se os comandos não aparecerem

Se você abriu o claude antes do CLAUDE.md existir, saia com /exit e abra claude de novo para ele carregar as regras.

PASSO 05Faça a primeira ingestão

No Obsidian, crie uma nota dentro de raw/artigos/ e cole um texto (um artigo curto ou suas anotações). Salve. No Claude Code, rode o comando de ingestão.

Claude Code
/ingest raw/artigos/nome-do-arquivo.md

Detalhamento: abra o Obsidian ao lado, veja as páginas novas aparecerem, clique num [[link]] e abra a Graph view — as conexões nascendo. Este é o momento "uau".

PASSO 06Faça a primeira pergunta

Ainda no Claude Code, pergunte algo sobre a fonte que você ingeriu.

Claude Code
/query qual a ideia central dessa fonte e por que ela importa?

Detalhamento: repare que a resposta vem com citações [[links]] e do wiki que ELE construiu — não do arquivo cru. É isso que acumula.

PASSO 07Conecte ao GitHub e sincronize

Crie uma conta em github.com (se não tiver) e um repositório privado (nunca público — é seu conhecimento pessoal). Suba a vault com os comandos abaixo.

Terminal — subir a vault
git init -b main git add . git commit -m "Primeiro commit do meu cerebro" git remote add origin URL-DO-SEU-REPOSITORIO git push -u origin main

Detalhamento: depois instale o plugin Obsidian Git (Settings → Community plugins → Browse → "Obsidian Git") e configure backup automático a cada 10 minutos. Pronto: seu cérebro faz backup sozinho e abre em qualquer máquina.

Atividade Prática
03. Comandos do dia a dia
Os três verbos que você vai usar sempre

Fluxo do dia a dia

  • Jogue uma fonte em raw/
  • Abra o Claude Code na pasta da vault (claude)
  • /ingest raw/... — leia os takeaways, deixe ele escrever
  • Veja no Obsidian (graph view)
  • /query quando precisar consultar · /lint semanalmente
ComandoO que faz
/ingest <arquivo>Adiciona uma fonte ao cérebro
/query <pergunta>Pergunta, responde com citações
/lintFaxina e saúde do wiki
O hábito é o segredo

Ingira 1 fonte por dia por uma semana. Em 7 fontes você já sente o acúmulo. Quando uma resposta for boa, peça para ele salvar como página.

04. Você sai com
O que está funcionando ao final desta aula

Você sai desta atividade com

  • Uma vault Obsidian estruturada (via bootstrap)
  • CLAUDE.md + comandos /ingest /query /lint
  • Primeira fonte ingerida e conectada
  • Primeira pergunta respondida com citações
  • Tudo sincronizado e com backup no GitHub
